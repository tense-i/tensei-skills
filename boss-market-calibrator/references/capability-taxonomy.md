# Capability Taxonomy

Code each valid JD using the categories below.

## Technical stack

Languages, frameworks, infrastructure, databases, middleware, cloud, tooling.

Examples:

- Java, Go, Python, TypeScript
- Spring Boot, Gin, React, Vue
- Redis, MySQL, PostgreSQL, Elasticsearch
- Kafka, RocketMQ, RabbitMQ
- Docker, Kubernetes, cloud platforms

## Business scenario

The business problem context the role is hired to solve.

Examples:

- high-concurrency transaction system
- payment and settlement
- risk control
- search/recommendation
- AI agent productization
- low-code editor
- IoT device management
- game economy or anti-cheat

## Engineering capability

Durable engineering abilities beyond tools.

Examples:

- system design
- performance optimization
- distributed consistency
- observability
- reliability and disaster recovery
- security and permissions
- testing and CI/CD
- cost optimization

## AI/data capability

Use when relevant.

Examples:

- LLM application development
- RAG
- tool calling
- agent orchestration
- prompt/system design
- model evaluation
- data pipeline
- feature engineering
- experimentation

## Soft and organizational capability

Examples:

- cross-team collaboration
- product sense
- ownership
- stakeholder communication
- requirement clarification
- mentoring
- technical leadership

## Capability layers

### L1 Entry-ticket capabilities

Baseline requirements needed to pass screening.

### L2 Core role capabilities

Capabilities that explain why the company hires this role.

### L3 Differentiators

Capabilities likely to improve seniority, salary, or competitiveness.

### L4 Noisy / low-priority requirements

Generic or weakly connected requirements that should not drive the user's learning plan.
