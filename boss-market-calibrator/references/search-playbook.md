# Search Playbook

## Search goals

Find 8-12 valid JDs from the last 30 days. Six valid JDs is the minimum for strong conclusions.

## Source priority

1. Recruiting platforms relevant to the target region.
2. Company career pages.
3. Search engine queries combining role, location, and recency.
4. User-provided JD text or screenshots when available.

## Expansion rounds

### Round 1: keyword expansion

Use synonyms, bilingual terms, and adjacent phrases.

Example for AI Agent engineer:

- AI Agent engineer
- LLM application engineer
- 大模型应用工程师
- Agent 工程师
- AI 应用后端
- LLM platform engineer

### Round 2: source expansion

Search company career pages and region-specific recruiting platforms.

### Round 3: adjacent role expansion

Expand to adjacent roles only when the capability profile remains close.

Example:

- AI Agent engineer -> LLM application engineer -> AI platform engineer
- high-concurrency backend -> transaction backend -> order/payment backend
- low-code frontend -> visual editor frontend -> complex web app frontend

## Do not over-expand

Do not include roles that shift the target too far. For example, do not mix AI research scientist JDs into an AI application engineer market report unless the user explicitly asks for both.
