# GitHub Trending Quick Deep Analyzer Output Template

对每个仓库逐个套用下面模板，禁止改成“前详后略”。

## （可选）仓库清单

- 1) `OWNER/REPO`
- 2) `OWNER/REPO`

## 仓库 #<序号>：<项目名>（一句话定位）

元信息：
- 信息抓取日期：YYYY-MM-DD
- Repo：`OWNER/REPO`
- 一句话定位
- 主要形态：库 / CLI / 框架 / 应用 / 模板 / 服务端 / SDK / 模型 / 数据集

### A. 详细 TL;DR

至少 8 条：
1. 它是什么
2. 为什么会上 Trending
3. 核心价值
4. 最适合谁
5. 典型使用场景：至少 3 个
6. 你用它的最短路径
7. 替代方案/竞品方向：至少 2 个
8. 主要风险/限制：至少 2 条

### B. 核心特性

至少 6 条，每条都按下面三段式写：

**特性 <n>：<名称>**
- 能做什么
- 适用场景
- 最小示例：命令 / 配置 / 伪代码三选一

如果没有现成示例，可写 `【示意】`，但不要省略。

### C. 解决了什么问题

至少 3 组，每组都要写：

**问题组 <n>**
- 痛点：传统方式哪里卡
- 方案：项目怎么解决
- 效果：可观察变化
- 验证方式/指标：如何验证

### D. 证据来源

列出实际读取过的入口：
- README
- Docs/Website
- Releases/Changelog
- Issues/PR
- Trending 页面

## Final check

- 每个仓库都有 A/B/C/D
- TL;DR 至少 8 条
- 特性至少 6 条，且每条有场景和示例
- 问题组至少 3 组，且每组有验证方式
- 写明抓取日期
- 不确定内容已标注
