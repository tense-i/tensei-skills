# Source Priority

Use current public evidence when possible.

## Hiring demand sources
1. official company career pages
2. official campus recruiting pages
3. large recruitment platforms and public JDs
4. credible technical communities discussing hiring expectations

## Interview style sources
1. nowcoder interview experiences and written test posts
2. public interview question collections with detailed answers
3. candidate reports from recent recruiting seasons
4. technical blogs describing common interview questions

## System depth sources
1. official engineering blogs
2. cloud provider reference architectures
3. mature open-source project docs
4. incident reports and postmortems
5. high-quality system design articles

## Evidence use
- Use JDs to infer what topics are demanded.
- Use interview posts to infer question style and wording.
- Use engineering sources to deepen synthesized production questions.
- Do not overclaim company-specific questions without direct support.
