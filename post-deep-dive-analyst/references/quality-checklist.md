# 质量检查清单

> 在输出深度分析文档前，必须逐项检查以下内容。任何一项未通过，必须补充完善后再输出。

---

## A. 覆盖完整性

- [ ] **P 覆盖**：正文每条核心观点（P1..Pn）是否都在文档中被逐条分析？
- [ ] **C 覆盖**：评论区有价值观点（C1..Cm）是否都被提炼和分析？
- [ ] **无遗漏**：是否有被跳过/合并/弱化的观点？（不允许"偷懒合并"）
- [ ] **链接完整**：每条分析是否能回溯到原文/评论的具体表述？

## B. 长度与密度

- [ ] **最小长度**：输出字数 O ≥ max(S × 5, 4000)？
  - 如未达到，是否按 Part 1/Part 2/... 分段输出？
- [ ] **信息密度**：每段新增内容是否都引入了"新信息维度"？
  - 新信息维度包括：解释/机制/边界/指标/步骤/案例/数据/反例
- [ ] **反套话**：是否存在空泛的过渡句/总结句可以删掉或替换为实质内容？
  - ❌ "这是一个值得深思的问题"
  - ❌ "总之，这个观点有其合理性"
  - ✅ 替换为具体的分析、数据或案例

## C. 检索与引用

- [ ] **检索充分**：是否针对核心概念/观点做了资料检索？
- [ ] **来源标注**：引用的数据/案例/研究是否标注了来源？
- [ ] **待核验标记**：不确定的信息是否标注了"待核验"并给出核验路径？
- [ ] **正反兼备**：是否检索了支撑和反驳两方面的资料？
- [ ] **来源汇总**：文档末尾是否有完整的"检索资料来源汇总"表？

## D. 分析深度

### D1. 观点解码
- [ ] 是否推断了每条观点的证据来源类型？
- [ ] 是否列出了作者的隐含假设？
- [ ] 是否还原了作者的认知模型（目标函数/系统切分/敏感变量）？
- [ ] 是否列出了前置知识清单？

### D2. 深度扩展
- [ ] 每条观点是否至少补齐了 5 个扩展模块？
  - 定义/机制/条件/反例/指标/案例/操作/数据
- [ ] 每个扩展是否能回答"这段新增内容带来了什么新维度"？

### D3. 评论区分析
- [ ] 是否绘制了评论区情绪图谱？
- [ ] 是否提炼了高价值评论？
- [ ] 是否归纳了争议焦点并分析了双方依据？
- [ ] 是否识别了评论区的信息偏差？

### D4. 批判性分析
- [ ] 是否给出了至少 2 条强反例？
- [ ] 是否给出了至少 3 条适用边界？
- [ ] 是否给出了至少 2 条风险提示？
- [ ] 是否给出了至少 2 条可证伪信号？
- [ ] 是否列出了主流对立观点？

### D5. 方法论逆向
- [ ] 是否还原了可重复的流程/方法论？
- [ ] 是否输出了至少 2 种"可执行结构"（决策树/清单/指标/流程图）？

## E. 保真度

- [ ] **不编造**：是否避免了虚构作者的具体经历/背景？
- [ ] **不猜测**：不确定的事实是否标注了"推断"或"待核验"？
- [ ] **三层区分**：是否明确区分了"作者原意""合理推断""扩展分析"？
- [ ] **保真模式**：是否按用户指定的 fidelity_mode 执行？
  - strict: 只扩展解释与通用知识
  - balanced: 可补充行业通用案例（不虚构作者经历）
  - creative: 可用类比与故事（不编造作者事实）

## F. 可行动性

- [ ] 是否有分级行动清单？（最小可行/进阶/系统级）
- [ ] 是否有常见误区 FAQ？
- [ ] 是否有延伸阅读推荐？
- [ ] 读者读完是否能"做决定/做行动/做复盘"？

## G. 客观性

- [ ] 批判性分析是否正反兼顾？
- [ ] 是否避免了"一边倒"的价值判断？
- [ ] 对立观点是否给予了公平的篇幅？
- [ ] 是否标注了分析者自身的立场/局限？
