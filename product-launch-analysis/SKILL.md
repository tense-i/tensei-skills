# Product Launch Analysis

用这个 skill 把产品材料整理成一份立项评审风格的报告，而不是泛泛的产品介绍。

## Core goal

围绕“这个产品值不值得做/投/继续验证”给出结构化判断，重点覆盖：
- 产品与用户场景
- 创新点与竞品矩阵
- 商业价值与 ROI
- 付费意愿与定价打包
- 风险、反方观点和两周验证计划

## Input expectations

兼容最小输入：
- 产品名
- 链接或一段介绍
- 任意证据材料：官网文案、评论、推文、访谈、价格页、社区讨论

信息不足时不要反复追问，直接降级输出并标记：
- `【证据】`
- `【推断】`
- `【假设】`

## Required workflow

1. 先校准产品定义、目标用户和核心价值主张。
2. 提炼至少 3 个使用场景和 3 个创新点。
3. 做直接竞品、间接竞品和替代方案分层。
4. 输出竞品矩阵、ROI 表和 WTP 分析。
5. 给出尖锐但可复盘的风险与反方判断。
6. 最后落到两周验证计划和评分卡。

## Grounding rules

- 不编造融资额、营收、用户量、具体价格、排名等未提供数据。
- 关键结论必须标 `【证据】`、`【推断】` 或 `【假设】`。
- 没有公开证据时，输出“缺证清单 + 如何补证据”。
- 锐评允许尖锐，但必须能回到机制、证据或验证方案。

## Output

按 `references/report-template.md` 输出固定章节。

必须包含：
- 立项结论摘要
- 创新点 Ledger
- 竞品矩阵
- ROI 表
- WTP 分析
- 风险清单
- Critical Take
- 两周验证计划
- 评分卡

## Quality bar

- 不要把“产品介绍”冒充“立项分析”。
- 每个关键判断都要有依据标签。
- 必须同时给支持理由和反对理由。
- 结论要能直接进入评审会或 PRD 前置调研。
